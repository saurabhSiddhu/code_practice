# 🧠 TWO-SUM - Insights

> **Solved on:** Friday, December 27, 2024

## 🎯 Mental Model

### Core Breakthrough
**Using hash map to store value -> index mapping transforms O(n²) search into O(1) lookup by trading space for time**

### Intuitive Explanation
Instead of asking 'does this other number exist somewhere?', we ask 'have I seen the complement of this number before?' This mental shift from searching forward to remembering backward is the core breakthrough

### Mental Picture/Analogy
Like a dance partner matching service - instead of checking every person to see if they're compatible with the current person, you remember everyone's preferences and instantly know if someone's ideal match has already arrived

### Common Pitfall to Avoid
Trying to use nested loops or sorting first, not realizing that hash map approach gives instant complement lookup while maintaining original indices

## ⚡ Constraint Analysis

### Critical Constraint
**Needing to return actual indices, not just whether solution exists - this ruled out sorting approaches and made hash map natural**

### Scalability Considerations
- **Breaking point:** Solution starts to struggle at around 10^7-10^8 elements due to memory constraints from hash map, but O(n) time complexity scales beautifully
- **Alternative approach:** If couldn't use extra space, would sort first then use two pointers, but would lose original indices and need extra work to recover them
- **Pattern:** Classic 'space-time tradeoff' pattern where we accept O(n) space to achieve O(n) time
