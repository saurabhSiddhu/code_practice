# 🧠 COIN-CHANGE-1 - Insights

## 🎯 Mental Model

### Core Breakthrough
**This is a classic 'minimum ways' DP where we build solutions bottom-up by considering each coin denomination and asking 'what's the minimum coins needed if I use this coin?'**

### Intuitive Explanation
For each amount, try every coin and see which one leads to the minimum total coins. The key insight is that dp[amount] = min(dp[amount-coin] + 1) for all valid coins

### Mental Picture/Analogy
Like making change at a store - for any amount, you check which bills/coins you have and pick the combination that uses the fewest total pieces

### Common Pitfall to Avoid
Trying to use greedy approach (always pick largest coin first) which doesn't work when coin denominations aren't canonical (e.g., coins=[1,3,4], amount=6)

## ⚡ Constraint Analysis

### Critical Constraint
**The constraint that coins can be reused unlimited times makes this an unbounded knapsack variant, not 0/1 knapsack**

### Scalability Considerations
- **Breaking point:** Solution scales as O(amount * coins.length) which becomes problematic when amount > 10^4 and many coin types
- **Alternative approach:** For very large amounts, could use BFS from target amount down to 0, but DP bottom-up is generally more intuitive
- **Pattern:** Unlimited use + minimum optimization = unbounded DP pattern
